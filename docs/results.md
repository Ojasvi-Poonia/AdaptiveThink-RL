## 1. KPI methodology

### 1.1 The bar (problem statement)

Improve **≥ 2 of 3** benchmarks by **≥ +5 percentage points** over the **same base
model** baseline, where each improved benchmark must also clear its absolute
target. Efficiency / latency is a secondary judged axis.

| Benchmark | Absolute target | Role |
|---|---|---|
| GSM8K | ≥ 50% | **IMPROVE** (primary +5% lever) |
| StrategyQA | ≥ 65% | **IMPROVE** (primary +5% lever) |
| MMLU | ≥ 45% | **MAINTAIN** (knowledge MC; never trained on) |

Targets and the +5% rule are encoded in `eval/plots.py`:

```python
KPI_TARGETS = {"gsm8k": 0.50, "mmlu": 0.45, "strategyqa": 0.65}
MIN_DELTA   = 0.05   # +5 percentage points
# met iff  (router − baseline) >= MIN_DELTA  AND  router >= target
# overall PASS iff n_met >= ceil(n_comparable * 2/3)   (n=3 -> need 2)
```

### 1.2 Exact baseline definition (apples-to-apples)

The **baseline** and the **trained** model are evaluated through the **identical
harness** (`eval/run_benchmarks.py`) — same datasets, same item subsample per
seed, same prompt template, same answer matcher
(`src/adaptivethink/router/reward.py::extract_answer` + `_answers_match`, reused
verbatim in training, CCDD, and eval), same decoding (greedy Pass@1).

- **Baseline = the SAME `Qwen/Qwen2.5-1.5B-Instruct`, NO adapter.** Two fixed
  policies anchor the Pareto plot:
  - `--route-mode always_think` → full CoT every question. **This is the KPI baseline anchor** (see `run_all_24h.sh` STAGE 5, `--baseline results/baseline_always_think.json`).
  - `--route-mode never_think` → direct answer every question (cheap-compute anchor).
- **Ours = base model + the GRPO router adapter** (`outputs/grpo-seed0`),
  `--route-mode model`, optionally verifier-gated (`--verifier-ckpt`). This is the
  run tagged `router` in `results/router.json`; `eval/plots.py` selects it by
  `tag == "router"`.

> The only thing that changes between baseline and ours is the **adapter +
> routing policy**. Datasets, few-shot/prompt, matcher, and decoding are held
> fixed, so the before/after delta is not confounded.

### 1.3 Multi-seed mean ± CI

RL is high-variance and the per-benchmark eval subsample (`--n 200` by default) is
small, so every reported Pass@1 is a **mean over seeds 0, 1, 2** with a reported
spread. Implementation: `eval/run_benchmarks.py::eval_benchmark_seeds` reshuffles
the `--n` subsample per seed and reports:

- `pass@1` — mean across seeds (the headline)
- `pass@1_std` — sample (Bessel-corrected, `/(N−1)`) standard deviation
- `pass@1_seeds` — the per-seed values, for transparency

**Reporting convention:** present mean ± std (n = 3 seeds). A 95% CI may be
approximated as `mean ± t·std/√n` (t≈4.30 for n=3, df=2) if a CI is preferred in
the deck; std is what the harness emits directly. When `--n` is unset (full set),
all seeds see identical items so std collapses to 0.0 — still reported honestly.

## 4. CCDD ablation (scaffold)

Isolates the novelty: same Dr.GRPO recipe, the only difference is whether the
CCDD self-difficulty curriculum filter is on. **CCDD-off** = train on the full
pool (no curriculum); **CCDD-on** = drop zero-gradient groups whose base-model
`solve_rate ∈ {0, 1}` (offline filter fed by `data/self_difficulty.jsonl`).

Produce the two runs:

```bash
# CCDD-ON  (curriculum filter from self-difficulty labels) — the default pipeline
CCDD=1 bash scripts/run_all_24h.sh          # -> outputs/grpo-seed0, results/router.json

# CCDD-OFF (plain RLVR, full pool, no curriculum)
CCDD=0 GRPO_OUT=outputs/grpo-noccdd-seed0 \
  bash scripts/run_all_24h.sh
python eval/run_benchmarks.py --benchmark all --model-name Qwen/Qwen2.5-1.5B-Instruct \
  --adapter outputs/grpo-noccdd-seed0 --route-mode model \
  --seeds 0,1,2 --tag router_noccdd --out results/router_noccdd.json --n 200
```

## 5. Exact reproduce commands

These map 1:1 to the **reproducibility video**. Two commands end-to-end.

```bash
# 0) one-time environment build (pinned stack: vllm 0.19.1 / torch 2.10 /
#    TRL 0.24 / Unsloth / transformers 4.57.6 + lm-eval). ~10-20 min.
./run.sh setup

# 1) the ONE 24h pipeline: baseline eval -> CCDD self-difficulty ->
#    Dr.GRPO RLVR -> self-distilled verifier -> multi-seed eval + KPI table ->
#    GGUF Q4_K_M export. Resumable (each stage skips if its output exists).
bash scripts/run_all_24h.sh
```

Useful overrides (all env-vars, see the `run_all_24h.sh` header for the full set
and the **time-budget cut order** if the wall-clock is tight):

```bash
SMOKE=1            bash scripts/run_all_24h.sh   # 50-step GRPO sanity first
MODEL=Qwen/Qwen2.5-3B-Instruct bash scripts/run_all_24h.sh   # scale up base
EVAL_SEEDS=0       bash scripts/run_all_24h.sh   # single-seed (faster eval)
EVAL_N=0           bash scripts/run_all_24h.sh   # full benchmark sets (slow)
```
